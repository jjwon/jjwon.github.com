Setting up a website using GitHub Pages

GitHub, well-known for its project repositories that serve millions of users, also has a useful service known as GitHub Pages that can serve static web pages hosted in a GitHub repository. It's a great place to host project documentation or personal web pages.

In order to set up the webpage, all you need to do is to create a new repo named username.github.com. For example, my website is hosted here. Afterwards, put a simple Hello World page here so GitHub creates your new page. In 10 minutes or so, your new GitHub page will be available at username.github.com.

If you want to link this new webpage to a custom domain (like www.jjwon.net), then there are a few more things to do. First, you will need to add a new file to the root of your GitHub repo called CNAME that contains the domain name you wish to use (e.g. jjwon.net). Second, you will need to update the DNS records for your custom domain. Add a new A record that points to 204.232.175.78, GitHub's IP, and a CNAME record with a value of username.github.io.

Afterwards, your website should be up and running. A lot of people like to use Jekyll, a Ruby-based static site generator, to help them create static sites. I chose to go with Pelican, a similar program that utilizes Python. Both of these tools are very useful, because you can write your content in Markdown or reStructuredText, and they will convert the content to web pages according to a template scheme.
